                                                   Step 5. Executing the Package 
          
Now that we have Configured the Packaged with updated files, its time to run.
1.     -In Visual Studio Click Cleaning and collecting part 1 and 2 Dataflow if your not already in the dataflow save and click start
2.     -When the Cleaning and collecting part 1 and 2 Dataflow completes the run, you should get green checks across the transformations.
3.     -Click into the control flow and Click into “Load error log to Database” Dataflow as it now has to run its cycle within the sequence container.  This dataflow takes the error log from the first run in the Cleaning and collecting part 1 Dataflow and Rescrubbing.
4.     -Once completed, you should have green checks across the transformations and E.T.L is now complete.

                           Step 7. Importing to Power BI
                               
Now that we have read and got read our data in SQL, its now time to import to Power BI for further data modeling and Reporting.
 
1.     -Open Power BI Desktop 
2.     -Select Get Data on the Pop-up Window
3.     -Select SQL Server Database in the list of sources and click connect 
4.     -Supply the Server name, which should be your local PC name, and the Database which would be “Dubai Real Estate”.
5.     -The data Connectivity mode should be set to import.
6.     -Click ok
7.     -In the Navigator window Select the “Dubai_RET” Table from the selection list to the left to preview. 
8.     -Select Transform Data to load Power query editor.




           Step 8. Transforming Data using Power Query Editor 



 Now that we have our data its time to transform and begin to build our data model that we are going to use for our report.
 
1.     -In Power Query Editor lets begin by scrolling to the “project_name_en” column click the header to select the column and select replace values in the transform section of the ribbon above the formula bar.
2.     -In the Replace values window, we need to replace the null values.
3.     -In the “Value To Find” slot, enter null
4.     -In the “Replace With” slot enter Not Listed
5.     -Click ok
6.     -Next, is the “master_project_en”, the column to the Right.
7.     -Like we did previously, click the header to select the column and select replace values in the transform section of the ribbon above the formula bar.
8.     -In the “Value To Find” slot, enter null
9.     -In the “Replace With” slot enter Not Listed
10.-click ok 



     Step 9. Transforming Data using Power Query Editor Pt 2
         

1.     -Next, is the "property_sub_type_en” scroll to the left to find the column.
2.     -Like we did previously, click the header to select the column and select replace values in the transform section of the ribbon above the formula bar.
3.     -In the “Value To Find” slot, enter null
4.     -In the “Replace With” slot enter Not Listed
5.     -Click ok
6.     -Scroll to the right to find column “actual_worth” and click the header 
7.     -click the icon in the left of the column header and change the data type to “fixed decimal number” 
8.     -Now we need to add a column. Head to the top left of the screen, click add column tab, and click custom column.
9.     -In the custom formula bar, insert “ [actual_worth]*[procedure_area] ” ensure the equal sign is present before the formula.
10.-In the New Column name bar above the formula, type Revenue and click ok.
11.-Like we did previously, click the icon in the left of the column header and change the data type to “fixed decimal number”


    Step 10. Transforming Data using Power Query Editor Pt 3 


       
1.     -Let’s scroll left and click the header for column area_name_en
2.     -Now that we have selected the column, we need to Duplicate the column by clicking Duplicate column in the add column tab in the general section of the ribbon.
3.     -Once its Duplicated, we need to reorder the columns. We achieve this by click and holding the area_name_en – copy column header and drag to the left although there is a better method.
4.     -once you drag the column to another position and let go, the table structure will appear in the formula bar. Go to the formula bar and click the arrow to the right end of the bar.
5.     -when the bar drops you should have a clear view of the column list. Scroll down in the list to find the "area_name_en - Copy" column.
6.     -highlight the column name with the quotations for the success of the M query. Once the column name is highlighted press Ctrl and C keys on your keyboard to copy it and backspace the name out with the comma that follows.
7.     -scroll up until you see the original column and insert the copy after the original in the list by pressing Ctrl and V keys to paste. Make sure to insert a comma to complete the list.
8.     -once you finish click the check mark to the left of the formula bar and click the arrow to the right of the formula bar to minimize the formula bar.

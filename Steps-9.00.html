Step 11. Transforming Data using Power Query Editor Pt 4



Now that we have copied the column and re-ordered the columns, we are now going to split this column to retrieve specific pieces of the values and add an additional 3 columns

1. -Make sure you are in the home tab and click the "area_name_en - Copy" column header.
2. -Once the Column header is selected, click Split column in the transform section of the ribbon. Once the drop-down menu appears click by delimiter.
3. -In the “Split Column Delimiter” settings window the delimiter in the “Select or enter delimiter” slot should be Comma. In the “Split at” section, select the Right-most delimiter which is the right most comma in the string text.
4. -Click ok.
5. -Now You should have a column named” area_name_en - Copy.2” present with “United Arab Emirates” as the value in the data.
6. -Double click the header to rename the column Country.

Step 13. Transforming Data using Power Query Editor Pt 6



Now we need to set up a Date refresh that would show in the report, the last time the report has been refreshed 
1.     -Make sure you are in the home tab and select New Source in the New Query section of the ribbon 
2.     -In the drop down select Blank Query.
3.     -Now we are going to use an M query function to pull the current date and time. Insert “= DateTime.LocalNow()” without the quotations it should display the current date and time.
4.     -In the top left corner of the screen click To Table and it should give you the Date and time as a value in Column1.
5.     -Right click over the column header and click Duplicate column.
6.     -Double click the “Column1 - Copy” column header to rename the column Date.
7.     -Like we did previously, click the icon in the left of the Date column header and change the data type to “Date”
8.     -Double click the “Column1 ” column header to rename the column DateTime.
9.     -To the Left in the Queries Tab, Double click the “Query1 ” header to rename the Query Date Refresh. 
10.-Go to the Home tab and click close and apply



Step 14. Adding Calendar table to Data Model


Now that we have loaded Our Data into the Model its time to add a Calendar table into our Data Model.
1.     -To the left there are three icons displayed vertically. Click the middle one to enter the Data view.
2.     -Ensure you are in the Home tab and select New table in the calculations section of the ribbon.
3.     -In the formula bar insert the following DAX Formula 
 
Calendar = ADDCOLUMNS(CALENDAR(MIN(Dubai_RET[instance_date]),MAX(Dubai_RET[instance_date])),
"DateAsInteger", FORMAT ( [Date], "YYYYMMDD" ),
 
"Year", YEAR ( [Date] ),
 
"Monthnumber", FORMAT ( [Date], "MM" ),
"YearMonthnumber", FORMAT ( [Date], "YYYY/MM" ),
 
"YearMonthShort", FORMAT ( [Date], "YYYY/mmm" ),
 
"MonthNameShort", FORMAT ( [Date], "mmm" ),
 
"MonthNameLong", FORMAT ( [Date], "mmmm" ),
 
"DayOfWeekNumber", WEEKDAY ( [Date] ),
 
"DayOfWeek", FORMAT ( [Date], "dddd" ),
 
"DayOfWeekShort", FORMAT ( [Date], "ddd" ),
 
"Quarter", "Q" & FORMAT ( [Date], "Q" ),
 
"YearQuarter", FORMAT ( [Date], "YYYY" ) & "/Q" & FORMAT ( [Date], "Q" ))
 
4.     -Once the formula is inserted, click the check mark and the formula will generate the table.
5.     -Next we need to create a relationship between our tables. To the left there are three icons displayed vertically. Click the bottom one to enter the Model view.
6.     -In the model view click and hold the Date field from the Calendar table and drag it to the instance_date field in the Dubai_RET table. The relationship should be 1 to many. Calendar being 1.
